# VetDiary

Мобильное приложение для владельцев питомцев. Проект разработан в рамках Практической работы №4 (Использование карты и внешних API).

## Технологии
* Язык программирования: Java
* База данных: SQLite (ORM-библиотека Room)
* Картография: OSMDroid (OpenStreetMap)
* Сетевые запросы: HttpURLConnection, Open-Meteo API
* Пользовательский интерфейс: XML
* Среда разработки: Android Studio

## Руководство пользователя

### 1. Главный экран и виджет погоды (API)
При запуске приложения открывается главный экран. В правом верхнем углу расположен виджет текущей погоды.
* Данные о температуре загружаются автоматически в фоновом режиме через публичный сервис Open-Meteo API.
* Обновление данных происходит при каждом запуске главного экрана. Подключение требует активного интернет-соединения.

### 2. Модуль интерактивной карты
Для перехода к карте нажмите на иконку геолокации в нижней панели навигации (Bottom Navigation).
* Интерактивная карта (OpenStreetMap) позволяет масштабировать изображение с помощью жестов (pinch-to-zoom) или двойного касания.
* На карте отмечены маркеры ветеринарных клиник.
* При нажатии на любой из маркеров на экране появляется информационное окно (Snippet) с точным адресом и описанием выбранного учреждения.

### 3. Модуль истории болезни и прививок
Осуществляется через центральную иконку в нижней панели навигации. Доступно добавление, удаление и редактирование записей (данные сохраняются в локальную базу данных).
* Для редактирования записи нажмите на соответствующую карточку.
* Для удаления записи используйте долгое нажатие (Long Click) по карточке.

## Инструкция по сборке APK из командной строки
Для сборки установочного .apk файла средствами интерфейса командной строки выполните следующие шаги:

1. Откройте корневую папку проекта VetDiary в терминале.
2. Убедитесь, что у вас установлен JDK и прописаны переменные среды (JAVA_HOME).
3. Выполните команду сборки (в зависимости от вашей операционной системы):
   * Windows (PowerShell): .\gradlew assembleDebug
   * Windows (CMD): gradlew assembleDebug
   * macOS / Linux: ./gradlew assembleDebug
4. Дождитесь появления сообщения BUILD SUCCESSFUL.
5. Готовый установочный файл app-debug.apk будет находиться в директории проекта по пути:
   app/build/outputs/apk/debug/app-debug.apk
